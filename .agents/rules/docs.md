Project Rule: Whenever any new update or architecture-related changes are made in the project, the existing documentation folder must be updated for both frontend and backend, including maintaining and updating their respective architecture diagrams.

Diagram Creation Rule: Architecture diagrams for frontend and backend must be created only once initially, and then only updated (not recreated) whenever changes occur.

Diagram Creation: Create initial frontend and backend architecture diagrams and save them in the documentation folder if not available, only one time.

Documentation Updates: Update frontend and backend documentation files and modify existing architecture diagrams in the documentation folder based on the latest changes, only if already available diagram.
